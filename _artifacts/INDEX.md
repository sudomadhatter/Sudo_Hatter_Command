## Where artifacts go (placement rules — read before creating a session folder)
**Artifacts go where you WORK FROM (Daniel, 2026-06-25).** The deciding factor is the workspace you have open
(your cwd), not only what the work is about.
- **Working from the home base** (this repo is your cwd) → home-base `_artifacts/`:
  - **project work** → a per-project bucket `_artifacts/<project-folder-name>/<YYYY-MM-DD>_<slug>/`
    (e.g. `_artifacts/aviationChat-AGY/`, `_artifacts/clean-bmad-workspace/`; the bucket = the `Projects/<name>/` folder name).
  - **home-base / cross-project work** (routing, `.agents/` toolkit, multi-project) → `_artifacts/_home/<YYYY-MM-DD>_<slug>/`.
  - Either way, append a row to THIS ledger.
- **Working from inside a project** (`Projects/<name>/` is your cwd) → **follow THAT project's rules**:
  project-local `Projects/<name>/_artifacts/<YYYY-MM-DD>_<slug>/` + its own `active-context.md`/`INDEX.md`
  (not this ledger).
- **Finding a project's history:** check BOTH the home-base bucket `_artifacts/<project>/` (sessions run from
  here) AND the project-local `Projects/<name>/_artifacts/` (sessions run inside it).
- Stories → `<epic>/<story>/`; retired → `_archived/`.
- Each session folder carries: `implementation_plan.md` (approved before edits), `walkthrough.md`
  (+ "Your Actions" git command), `task-list.md`; add `code-review.md` / `self-audit-stress-test.md` when
  those run. Then update that workspace's `active-context.md` (+ a row in the matching `INDEX.md`).
